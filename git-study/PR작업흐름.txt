[PR(Pull Request) 정리]

1. PR이란?
- Pull Request는 내가 작업한 브랜치의 코드를 다른 브랜치(main)에 합쳐달라고 요청하는 것이다.

2. PR을 사용하는 이유
- 코드 리뷰를 위해
- 협업 시 코드 충돌을 줄이기 위해
- 작업 기록을 남기기 위해
- main 브랜치를 안정적으로 유지하기 위해

3. PR 기본 흐름
개인 브랜치 작업
→ git add
→ git commit
→ git push
→ GitHub에서 Compare & pull request
→ Create pull request
→ Merge pull request
→ Confirm merge

4. PR 한줄 정리
- "내 작업 브랜치의 코드를 main에 합쳐주세요" 라는 요청

--------------------------------------

[Git 작업 루틴 정리]

1. 작업 시작 전
- main으로 이동
- 최신 코드 가져오기

git checkout main
git pull origin main

2. 새 작업 시작
- main에서 새 브랜치 생성

git checkout -b feature/작업이름

3. 작업 후 저장
git add .
git commit -m "작업 내용"
git push origin feature/작업이름

4. GitHub에서 PR 생성
- Compare & pull request
- 제목/설명 작성
- Create pull request

5. 병합 후 로컬 최신화
git checkout main
git pull origin main

6. 필요 시 브랜치 정리
git branch -d feature/작업이름
git push origin --delete feature/작업이름

--------------------------------------

[오늘 배운 핵심]
- push = 내 코드를 원격 저장소에 올리기
- pull = 원격 저장소의 최신 내용을 가져와서 반영하기
- fetch = 최신 내용을 가져오기만 하고 바로 반영하지 않기
- 브랜치는 각자 작업 공간
- main은 최종 기준 브랜치
- 협업 기본 흐름은 브랜치 작업 → PR → main 병합
