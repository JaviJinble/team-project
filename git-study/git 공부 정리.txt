[수업 5일차 학습 정리]

1. Git 기본 흐름
- git add : 변경 파일 등록
- git commit : 작업 기록 저장
- git push : 원격 저장소에 업로드

기본 순서
수정 → add → commit → push

--------------------------------------

2. 협업 방식 (브랜치 전략)
- main : 최종 코드 (기준 브랜치)
- feature/* : 개인 작업 브랜치

협업 흐름
main → 브랜치 생성 → 작업 → commit → push → PR → merge

--------------------------------------

3. 브랜치 사용 이유
- 각자 독립적으로 작업 가능
- 충돌 최소화
- 코드 안정성 유지

--------------------------------------

4. 브랜치 명령어
브랜치 생성 및 이동
git checkout -b feature/bus --> 개인 작업 공간

브랜치 이동
git checkout feature/bus

--------------------------------------

5. Pull Request(PR)
- 내 코드를 main에 합쳐달라는 요청
- GitHub에서 생성
- 코드 리뷰 후 merge 진행

--------------------------------------

6. merge (병합)
- feature 브랜치 → main 합치기
- 팀원이 확인 후 진행

--------------------------------------

7. 충돌 (Conflict)
- 같은 파일을 여러 사람이 수정했을 때 발생

표시 형태
<<<<<<< HEAD
내 코드
=======
상대 코드
>>>>>>>

해결 방법
- 직접 수정 후
git add → git commit → git push

--------------------------------------

8. 오늘 겪은 에러 정리
1) 브랜치 없이 push → 에러
2) add 안 하고 commit → 에러
3) 원격이 더 최신 → pull 먼저 필요
4) merge 메시지(MERGE_MSG) 저장 필요

--------------------------------------

9. 협업 규칙
- main 직접 수정 금지
- 작업 전 git pull 필수
- 브랜치에서 작업
- PR 후 merge

--------------------------------------

[한줄 정리]
Git 협업 = 브랜치에서 작업 → PR → main 병합

-------------------------------------

[Git push / pull / fetch(패치) 정리]

1. git push
- 내 로컬 저장소의 커밋을 원격 저장소(GitHub)에 업로드

사용 상황
- 내가 작업한 내용을 팀원에게 공유할 때

명령어
git push origin 브랜치명

예시
git push origin main
git push origin feature/bus

한줄 정리
→ "내 코드 올리기"

--------------------------------------

2. git pull
- 원격 저장소의 최신 내용을 가져와서
- 내 로컬 코드와 자동으로 merge까지 수행

사용 상황
- 다른 사람이 작업한 내용을 내 코드에 반영할 때

명령어
git pull origin 브랜치명

예시
git pull origin main

한줄 정리
→ "가져오기 + 자동 병합"

--------------------------------------

3. git fetch (패치)
- 원격 저장소의 최신 내용을 가져오지만
- 내 코드에는 적용하지 않음 (merge 안함)

사용 상황
- 변경사항을 먼저 확인하고 싶을 때

명령어
git fetch origin

이후 확인
git log origin/main

한줄 정리
→ "가져오기만 하고 적용은 안함"

--------------------------------------

4. pull vs fetch 차이

pull = fetch + merge

즉,
git pull = git fetch + git merge

--------------------------------------

5. 흐름 비교

push
내 PC → GitHub

pull
GitHub → 내 PC + 자동 적용

fetch
GitHub → 내 PC (확인용, 적용X)

--------------------------------------

6. 협업에서의 사용 흐름

작업 시작 전
git pull origin main

작업 후
git add .
git commit -m "작업 내용"
git push origin feature/브랜치명

--------------------------------------

[최종 한줄 정리]

push = 올리기  
pull = 가져와서 적용  
fetch = 가져오기만 (확인용)
